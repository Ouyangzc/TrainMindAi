# 学员端 MVP 详细设计方案

## 1. 文档目的

本文档定义 TrainMind 学员端第一阶段的产品边界、信息架构、数据来源、权限规则、页面行为、接口需求和验收标准，作为产品评审、前后端拆分与测试验收的共同依据。

本文档承接：

- 《企业级知识助教系统落地方案》
- 《课程资料与知识库治理实施计划》
- 《课程资料上传与文件服务设计方案》
- 《知识库治理设计方案》

本阶段定位为“基于企业课程资料的 AI 学习与查阅空间”，不建设完整 LMS。

## 2. 已确认的产品决策

### 2.1 核心映射

不新增独立的学员课程实体，直接复用现有课程体系：

| 管理端实体 | 学员端含义 |
|---|---|
| `course` | 课程 |
| `course_module` | 课程目录中的模块/章节 |
| `course_document` | 资料管理对象 |
| `course_document_version` | 资料的具体文件版本 |
| `knowledge_base.current_version_id` | 当前对学员生效的知识库版本 |
| `knowledge_base_version_document` | 学员可见资料快照与 AI 检索范围 |
| `course_user` | 用户对课程的直接访问授权 |

关键原则：

```text
知识库是课程的 AI 能力载体，不是另一门课程。
模块是课程目录，不再重复建设章节表。
管理端最新资料不等于学员可见资料。
```

### 2.2 产品框架

学员端采用组合方向：

- B：课程学习空间作为整体框架。
- C：AI 学习助教作为课程内默认入口。
- A：资料库作为独立页面，并承接 AI 引用跳转。

### 2.3 MVP 非目标

第一阶段不实现：

- 培训班、班级选课和批次管理。
- 考试、题库、错题本和证书。
- 学分、强制课时和监管报表。
- 复杂章节前置关系和解锁条件。
- AI 自动生成并发布课程目录。
- 跨课程 AI 问答。
- 学员访问管理端草稿或历史知识库版本。

上述能力保留为后续阶段，不应提前污染 MVP 数据模型。

## 3. 用户与权限边界

### 3.1 目标用户

MVP 目标用户为具有有效 `course_user(student)` 授权的企业学员。

管理员、课程负责人和教师继续使用管理端，不通过学员端修改课程内容。

### 3.2 课程访问判定

学员可进入一门课程必须同时满足：

```text
course.del_flag = '0'
AND course.status = 'active'
AND course_user.del_flag = '0'
AND course_user.access_role = 'student'
AND course_user.access_status = 'active'
AND (course_user.start_at IS NULL OR start_at <= 当前时间)
AND (course_user.end_at IS NULL OR end_at >= 当前时间)
```

存在有效课程授权但没有已发布知识库版本时，可以显示课程卡片，但进入后显示“课程内容准备中”，不返回管理端资料，不允许 AI 问答。

### 3.3 权限校验位置

以下接口必须在服务端分别校验课程授权，不能仅依靠前端隐藏入口：

- 我的课程列表。
- 课程空间概览。
- 课程目录。
- 资料列表与搜索。
- 资料详情、预览和下载。
- AI 会话、提问和引用详情。
- 学习记录。

不得通过猜测 `courseId`、`documentId`、`versionId` 或对象存储地址访问未授权内容。

## 4. 信息架构

```text
学员端
├── 我的课程
│   ├── 进行中
│   ├── 尚未开始
│   └── 已到期/已停用（可选展示）
└── 课程空间
    ├── AI 学习助教（默认）
    ├── 课程目录
    ├── 资料库
    └── 学习记录（MVP 轻量）
```

课程空间顶部提供当前课程切换器。切换课程后，目录、资料、AI 会话和学习记录必须同步切换到新的 `courseId`，不得残留上一门课程上下文。

## 5. 核心数据一致性

学员在同一时间看到的内容必须来自同一个已发布知识库版本：

```text
课程目录中的可见资料
= 资料库中的资料
= AI 检索范围
= AI 引用可打开的资料版本
= knowledge_base.current_version_id 对应的资料快照
```

### 5.1 目录来源

课程目录来源于 `course_module`，但学员端只展示与当前发布知识库版本相关的有效模块。

推荐展示规则：

1. 按 `course_module.sort_order`、`id` 排序。
2. 模块必须未删除且状态为 `active`。
3. 模块至少包含一份当前发布快照中的资料时才展示。
4. 无 `module_id` 的资料归入“通用资料”，不作为编号章节。
5. 管理端模块说明可作为学员端章节简介。

不允许学员目录直接读取 `course_document.latest_version_id`，否则可能看到尚未发布的新文件版本。

### 5.2 资料来源

学员资料以 `knowledge_base_version_document.document_version_id` 为权威版本。资料标题、文件名、类型、页数和引用定位均应解析到该快照绑定的版本。

管理端的 `parsed` 仅表示文件已具备进入知识库的条件，不表示学员可见。

### 5.3 发布切换

知识库发布成功后，新请求读取新的 `current_version_id`。进行中的问答请求固定使用发起时的版本，避免回答和引用跨版本。

历史会话保存问答时使用的 `knowledge_base_version_id` 和引用版本信息。MVP 中如果旧版本不再允许学员打开，应提示“该引用来自历史版本”，并提供当前资料入口，不静默替换引用内容。

## 6. 页面设计

### 6.1 我的课程

#### 目标

让学员确认自己有哪些课程，并快速继续最近使用的课程。

#### 页面内容

- 页面标题与当前用户信息。
- 课程名称、简介、分类和负责人。
- 内容发布状态。
- 授权有效期。
- 最近访问时间。
- “进入课程”或“继续学习”操作。

#### 排序建议

1. 最近访问过的有效课程。
2. 其他有效课程，按 `course.sort_order` 排序。
3. 尚未开始或已到期课程放入次级分组。

#### 课程卡片状态

| 状态 | 条件 | 操作 |
|---|---|---|
| 可学习 | 授权有效且已有发布版 | 进入课程 |
| 内容准备中 | 授权有效但无发布版 | 禁止进入内容，显示说明 |
| 尚未开始 | `start_at` 晚于当前时间 | 展示生效时间 |
| 已到期 | `end_at` 早于当前时间 | 默认不可进入 |
| 已停用 | 课程或授权停用 | 不可进入 |

### 6.2 课程空间外壳

课程空间负责提供稳定的课程上下文，而不是单独承载内容。

固定区域：

- 当前课程名称及课程切换器。
- AI 学习助教、课程目录、资料库、学习记录导航。
- 知识库当前版本或“内容已更新”提示。
- 授权有效期提示。

默认打开 AI 学习助教。用户最后一次停留页面可作为后续优化，不纳入 MVP 必须项。

### 6.3 AI 学习助教

#### 目标

让学员用自然语言理解当前课程，并能验证答案依据。

#### 页面内容

- 当前课程和当前知识库版本。
- 新建会话。
- 推荐问题。
- 问答消息区。
- 回答引用卡片。
- 输入框与发送操作。
- 明确的知识范围说明。

#### 问答规则

1. 请求必须包含当前 `courseId`。
2. 业务后端校验课程授权并解析当前发布 `knowledge_base_version_id`。
3. AI 只检索该版本的有效 Chunk。
4. 回答必须携带引用；无充分证据时拒答或说明资料中未找到依据。
5. 不默认使用未标识的模型通用知识补全答案。
6. 每个引用至少包含资料标题、资料版本、页码或片段定位。
7. 点击引用进入资料库或预览器中的对应位置。

#### 异常状态

- 无发布版：提示课程内容尚未发布。
- 构建或发布新版本中：继续使用当前发布版。
- 检索无结果：提示未在当前课程资料中找到可靠依据。
- AI 服务不可用：保留资料库入口，不影响资料查阅。
- 授权到期：停止继续提问并引导联系管理员。

### 6.4 课程目录

#### 目标

以管理端模块顺序呈现课程内容结构。

#### 页面内容

- 模块名称和说明。
- 模块下当前已发布的资料。
- 文件类型和必要的内容简介。
- 最近访问标识。
- 手动“标记已完成”（若最终确认进入 MVP）。

MVP 不计算强制学习时长。若未确认完成规则，页面仅显示“最近访问”，不展示虚假的完成百分比。

### 6.5 资料库

#### 目标

支持学员主动搜索、筛选和核对课程原始资料。

#### 页面内容

- 关键词搜索。
- 模块筛选。
- 文件类型筛选。
- 资料名称、所属模块、文件类型和当前发布版本。
- 在线预览或下载操作。
- 从 AI 引用进入时的目标资料高亮与页码定位。

学员端不展示上传、解析、构建、失败原因等管理状态。

搜索范围只能是当前发布快照。即使管理端已有更新版本，搜索结果也不能提前返回。

### 6.6 学习记录

MVP 采用轻量行为记录，不宣称知识掌握度：

- 最近进入的课程。
- 最近访问的模块和资料。
- 最近 AI 会话。
- 可选的手动完成标记。

“阅读过”“完成”“掌握”必须是不同语义。仅打开资料不能自动认定为已经完成或掌握。

## 7. 关键业务流程

### 7.1 进入课程

```text
登录
-> 查询有效 course_user
-> 展示我的课程
-> 选择 courseId
-> 再次校验授权
-> 查询 current_version_id
-> 有发布版：进入 AI 学习助教
-> 无发布版：显示内容准备中
```

### 7.2 AI 提问与查看引用

```text
输入问题
-> 后端校验课程授权
-> 固定当前知识库版本
-> AI 检索并生成答案
-> 返回引用元数据
-> 点击引用
-> 再次校验课程与资料版本权限
-> 打开原文定位
```

### 7.3 管理端发布新版本

```text
管理员上传/解析资料
-> 创建知识库草稿
-> 选择资料版本快照
-> 构建成功
-> 人工发布
-> current_version_id 切换
-> 学员后续请求读取新版本
```

发布过程中，旧发布版继续服务，不向学员暴露半成品。

## 8. MVP 接口建议

接口路径为设计建议，最终可按现有后端规范调整。

### 8.1 课程

| 方法 | 路径 | 用途 |
|---|---|---|
| `GET` | `/student/courses` | 我的课程列表 |
| `GET` | `/student/courses/{courseId}` | 课程概览与访问状态 |
| `GET` | `/student/courses/{courseId}/outline` | 当前发布版课程目录 |

课程列表不能复用管理端 `/course/list` 后在前端过滤，必须由后端按当前用户授权查询。

### 8.2 资料

| 方法 | 路径 | 用途 |
|---|---|---|
| `GET` | `/student/courses/{courseId}/documents` | 当前发布快照资料列表 |
| `GET` | `/student/courses/{courseId}/documents/{documentId}` | 资料详情 |
| `GET` | `/student/courses/{courseId}/documents/{documentId}/preview` | 获取受控预览信息 |
| `GET` | `/student/courses/{courseId}/documents/{documentId}/download` | 下载当前发布资料版本 |

预览和下载是否开放由课程级策略控制。对象存储不应返回永久公共 URL，推荐使用短期签名地址或后端流式响应。

### 8.3 AI 会话

| 方法 | 路径 | 用途 |
|---|---|---|
| `GET` | `/student/courses/{courseId}/chat/sessions` | 当前课程会话列表 |
| `POST` | `/student/courses/{courseId}/chat/sessions` | 新建会话 |
| `GET` | `/student/courses/{courseId}/chat/sessions/{sessionId}` | 会话消息 |
| `POST` | `/student/courses/{courseId}/chat/sessions/{sessionId}/messages` | 提问 |
| `GET` | `/student/courses/{courseId}/citations/{citationId}` | 引用详情与定位信息 |

会话必须绑定用户、课程和知识库版本。服务端不信任前端直接传入的 `knowledge_base_version_id`，应根据课程当前发布状态决定，并记录最终使用版本。

### 8.4 学习记录

| 方法 | 路径 | 用途 |
|---|---|---|
| `GET` | `/student/courses/{courseId}/activities` | 最近学习活动 |
| `POST` | `/student/courses/{courseId}/activities` | 记录资料或模块访问 |

若第一版不展示学习记录，可以先保留服务端问答日志和最近访问字段，不提前建设完整进度表。

## 10. 状态与空页面

必须覆盖以下情况：

| 场景 | 页面行为 |
|---|---|
| 学员没有课程授权 | 我的课程空状态，引导联系管理员 |
| 授权尚未生效 | 显示开始时间，不允许进入 |
| 授权已到期 | 显示到期状态，不允许继续访问 |
| 课程已停用 | 显示课程暂停或从默认列表移除 |
| 没有发布知识库版本 | 显示内容准备中 |
| 发布版没有资料 | 显示暂无已发布资料，AI 不可用 |
| 模块无发布资料 | 不在学员目录显示该模块 |
| AI 没有检索到依据 | 明确拒答并推荐查阅资料库 |
| 引用资料已成为历史版本 | 提示历史引用，不静默换成新内容 |
| AI 服务故障 | 问答不可用，资料库仍可使用 |

## 11. 管理端配套要求

现有课程管理结构保持：

```text
基本信息 / 模块管理 / 资料管理 / 知识库治理 / 课程成员
```

为保证学员端结果可控，建议补充：

1. 在模块管理中提示模块名称、说明和排序会直接展示给学员。
2. 资料列表区分“已解析”和“已在当前发布版中”。
3. 课程列表区分课程运营状态与知识库发布状态。
4. 增加“学员端预览”，以当前发布版只读展示目录和资料。
5. 有效学员数统计应考虑授权状态及起止时间。
6. 课程分类改为数据字典或受控配置，不在前端写死。

## 12. 安全与审计要求

- 所有查询必须携带租户边界。
- 所有学生接口从登录态获取 `userId`，不能由前端指定访问用户。
- 下载和预览必须校验当前发布快照中的具体资料版本。
- AI 请求记录用户、课程、知识库版本、会话、问题和引用。
- 不在前端暴露对象存储长期凭据。
- 禁止从管理端接口返回未发布资料后由前端自行过滤。
- 授权失效后，历史会话列表和引用访问按同一课程权限立即失效。

## 13. MVP 实施拆分

### 阶段一：学员课程入口

1. 实现学生课程授权查询。
2. 实现“我的课程”接口和页面。
3. 实现课程空间外壳与课程切换。
4. 覆盖无授权、未开始、已到期、无发布版状态。

### 阶段二：目录与资料库

1. 基于当前发布快照组装课程目录。
2. 实现学员资料列表、搜索和筛选。
3. 实现受控下载。
4. 明确在线预览支持范围；若暂不实现预览，先支持下载和引用定位信息。

### 阶段三：AI 学习助教

1. 实现课程内会话。
2. 严格绑定当前发布知识库版本。
3. 返回结构化引用。
4. 实现无依据拒答和服务异常降级。
5. 从引用跳转资料详情。

### 阶段四：轻量学习记录

1. 记录最近课程、模块和资料访问。
2. 展示最近活动。
3. 是否加入手动完成标记，待产品确认后实施。

## 14. MVP 验收标准

### 14.1 多课程与权限

- 学员只能看到自己有效授权的课程。
- 一个学员被授权多门课程时，课程列表和切换器展示完整且互不串数据。
- 修改、停用或到期授权后，直接访问旧链接也被拒绝。
- 不同租户的课程、资料和会话完全隔离。

### 14.2 目录与资料

- 模块顺序与管理端排序一致。
- 目录和资料库只包含当前发布知识库版本的资料。
- 管理端上传或解析新版本不会立即改变学员内容。
- 发布新知识库版本后，学员新请求切换到新快照。
- AI 引用打开的是回答时实际使用的资料版本或明确的历史版本提示。

### 14.3 AI 问答

- 每次问答限定当前课程及知识库版本。
- 有依据的回答展示可操作引用。
- 无充分依据时不编造课程结论。
- 切换课程后，新问题不得召回上一课程内容。
- AI 服务不可用时，资料库仍然可访问。

### 14.4 用户体验

- 学员可以在三次操作内从登录进入任一课程的 AI 提问区。
- 可以从 AI 引用直接找到对应资料和页码/章节。
- 所有空状态说明原因，并给出可执行的下一步。
- 桌面端为主要验收环境，同时保证窄屏下完成课程切换、提问和资料查阅。

## 15. 待产品确认项

以下问题不阻塞课程入口、目录、资料库和 AI 问答的设计，但实施对应功能前必须确认：

1. 第一版是否展示学习进度；如果展示，完成动作如何定义。
2. 是否允许学员手动标记模块或资料为已完成。
3. 资料是否默认允许下载，还是由课程级字段控制。
4. 在线预览第一版需要支持哪些格式。
5. 历史知识库版本是否允许具有当前课程权限的学员继续访问。
6. 学员首页默认停留在“我的课程”还是自动进入最近课程。
7. AI 无课程依据时是否绝对拒答，或允许明确标记的通用知识补充。
8. 是否展示已到期课程及历史学习记录。
9. 是否需要移动端优先设计。

在这些问题确认前，本文档采用以下默认值：

- 不展示具有考核含义的完成进度，只记录最近访问。
- 不自动将打开资料判定为完成。
- AI 无可靠课程依据时拒答。
- 不支持跨课程问答。
- 学员首页先展示“我的课程”。
- 桌面端优先，提供基础响应式适配。

## 16. 后续演进边界

当业务明确需要正式培训、考核和掌握度分析时，再引入：

```text
培训班/批次
-> 班级课程关系
-> 章节或知识点学习任务
-> 题库与考试
-> 作答与错题
-> 知识点掌握度
-> 培训完成与证书
```

即使进入该阶段，现有 `course -> module -> document -> knowledge base version` 仍可作为课程内容主干，不需要推翻；新增模型负责教学组织和学习结果，不与知识库内容治理重复。
